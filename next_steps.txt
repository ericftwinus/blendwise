1. HIPAA COMPLIANCE (currently claimed but not real)
   - PHI (diagnoses, symptoms, allergies, nutrient targets) is sent to xAI
     Grok API with NO Business Associate Agreement (BAA)
   - xAI does not offer a HIPAA BAA program — need a HIPAA-eligible LLM
     provider (e.g., Azure OpenAI, AWS Bedrock, or a self-hosted model)
   - No audit logging — no record of who accessed/modified patient data
     (HIPAA Technical Safeguard requirement)
   - HIPAA Notice of Privacy Practices page is a dead link (href="#")
   - Terms of Service and Privacy Policy pages are dead links (href="#")
   - Need a BAA with Supabase (they offer one on Pro/Enterprise plans)
   - Need breach notification procedures documented
   - Need data encryption verification (Supabase handles at-rest, but
     should be verified and documented)

2. PAYMENT / STRIPE INTEGRATION (completely absent)
   - No Stripe SDK, no checkout flow, no webhook handler
   - Landing page shows Tier 2 ($39/mo) and Tier 3 ($79/mo) but clicking
     any pricing CTA just goes to /signup — no tier selection
   - subscription_tier column in profiles defaults to 1 and is never
     updated by any code
   - "Upgrade" buttons in settings and grocery delivery CTA are dead
   - Need: Stripe integration, checkout flow, subscription management,
     webhook handler for payment events, tier upgrade/downgrade logic

3. SERVICE ROLE KEY MISCONFIGURATION
   - .env.local has SUPABASE_SERVICE_ROLE_KEY set to the anon/publishable
     key (not the actual service role key)
   - Server-side API routes use NEXT_PUBLIC_SUPABASE_ANON_KEY, meaning
     privileged operations run under the anon key
   - The real service role key is in .env under SUPABASE_SERVICE_ROLE_KEY
   - Need to fix .env.local and ensure API routes use the correct key
     for server-side operations
